3. 03 - Multimodal & Context Management#

This chapter introduces visual elements and contextual memory management, features that are absolutely crucial in delivering complex requirements.

3.1. 1. Version Rewind (Rewind)#

If you feel that a newly added feature is redundant or poorly implemented:

  • Type /rewind or rapidly press the ESC key twice to trigger the version rewind menu.

  • A restore point is created every time an input request and its edits are completed.

  • You can select “Restore code and session,” and the system will roll the codebase back to that specific moment.

Note: Files generated by external terminal commands (like the node_modules folder created by npm install, or other miscellaneous files) cannot be rewound. It is highly recommended to use Git alongside this system for more reliable rollbacks.

3.2. 2. Image Multimodal Input#

Tired of describing UI purely with text? You can provide Claude Code directly with your design mockups!

  • Drag and Drop: Drag the image directly into your active terminal/Claude Code interface.

  • Clipboard Paste: Copy the image and use Ctrl + V to paste it. (Take note: Even on macOS, you must use Ctrl + V, not Command + V).

Image Suggestion (Nanobanana Prompt): A 3D isometric masterpiece illustrating a user physically dragging a highly detailed, vibrant mobile app UI mockup design (showing buttons, colorful charts, and text) into a glowing, translucent holographic terminal console. The terminal console is breaking down the image into floating lines of glowing green and blue code chunks. The scene is filled with magical, floating digital particles, shot in an isometric angle, soft ambient lighting mixed with bright neon digital bursts, unreal engine 5 render, highly detailed modeling.

3.3. 3. Installing MCP Servers (Using Figma as an Example)#

To ensure the code rendition of your front-end design mockup is pixel-perfect down to the font weight and padding, you can integrate external tools via the MCP (Model Context Protocol):

  1. Exit Claude Code (Ctrl + C), and run the official Figma MCP installation command (e.g., using npx ...).

  2. Inside Claude Code, type /mcp, and you will see the newly installed Figma Server.

  3. If it requires authentication, select Authenticate and grant access permissions in the pop-up browser window.

3.4. 4. Recovering Historical Sessions (Resume)#

Because you closed the terminal earlier, your previous chat history was lost. How do you find it?

  • Type the /resume command, choose the most recent session from the list, and press Enter to restore.

  • The simpler way: Start Claude Code with the claude -c flag (‘Continue’), which will automatically restore the latest recent session.

3.5. 5. Using MCP to Translate Design Mocks into Code#

Once authenticated:

  1. Issue a prompt to Claude Code: “Modify the page to match the Figma mockup strictly.”

  2. Attach the Figma object link obtained via Copy link to selection, then hit Enter.

  3. The model will automatically identify and call the relevant MCP Tool (e.g., get_design_context).

  4. It will systematically invoke screenshot tools to analyze padding, margins, dimensions, and styling metadata. Ultimately, it produces a front-end UI that is a 1:1 match to the design!

3.6. 6. Context Compaction and Clearing#

After parsing images, executing commands, and calling tools, the LLM’s context window can become exceedingly crowded.

  • Clear Context: Use /clear. This completely eradicates the memory of the current chat, creating a brand new slate of understanding.

  • Compact Context: Use /compact. The system intelligently summarizes the development focus, decisions, and outcomes of the current cycle, discarding massive intermediate data bits (like Image Base64 strings or lengthy error logs). This drastically restores response speed and lowers your Token expenditure. You can also use Ctrl + O to unfold and read the post-compacted context result.

3.7. 7. Project Memory Files (CLAUDE.md)#

Beyond the context of a single session, how do you pass “unified project settings” across multiple sessions?

  • CLAUDE.md: You can have Claude instantly read this file upon boot as a permanent “plug-in” Prompt.

  • Generation Method: Use /init to automatically generate a standard guideline file for the current project, or simply create it manually.

  • Edit/Locate config: Use /memory to check whether the .md file is stored at the project level (current directory) or system level (user global directory). You can hardcode rules at the end of the file, such as: “Add ‘Happy Coding’ to the end of every response you make.” Once saved, every subsequent interaction will abide by this convention.

3.8. Knowledge Quiz#

Q1: What shortcut MUST be used to paste an image within Claude Code on MacOS?

Answer Even on macOS, you must use `Ctrl + V` to paste an image. `Command + V` will not work.

Q2: Why is using Figma MCP highly recommended over simply sending the design mockup image directly to Claude Code?

Answer Simply reading an image often leads to losing exact font sizes, precise padding, and exact margin details. An MCP Tool extracts the exact metadata, measurements, and contextual sliced images directly from Figma's raw data, ensuring a high-fidelity 1:1 code reproduction.
